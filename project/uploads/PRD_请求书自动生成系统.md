# 请求书（請求書）自动生成系统 · PRD v2

> **本 PRD 给 Claude Design 用**：按"软件定位 → 业务规则 → 流程 → 页面 → Excel 模板 1:1 复刻 → 组件规范"的顺序生成可点击高保真原型。
> **唯一输出标准**：系统**最终交付物 = PDF 文件**，每份 PDF 必须与参考 Excel 模板 `请求书自动导出 门店信息参考.xlsx` 视觉 1:1 一致。
> **目标上线**：2026 年 6 月（V1.0 简化版）

---

## 0. 软件定位

### 0.1 一句话
> **一个内部工具，让运营每月初通过"选店铺 + 填金额"两步，批量生成日文请求书 PDF（输出格式 1:1 还原 Excel 模板），并保留 PDF 历史记录。**

### 0.2 最终输出物：PDF（强约束）
| 项目 | 规范 |
| --- | --- |
| 文件类型 | `.pdf`（唯一） |
| 单份大小 | A4 纵向 210×297mm，单页 |
| 字体嵌入 | 必须嵌入日文字体（Noto Sans JP 或 Yu Gothic），不允许字体回退导致乱码 |
| 边距 | 上下 20mm，左右 18mm |
| 命名规则 | `{请求书编号}.pdf`，例 `202604-STA-01.pdf` |
| 批量打包 | N ≥ 2 时打包成 `请求书_{YYYY年M月}_{N}份_{生成时间戳}.zip` |
| 打印兼容 | 支持 A4 纸直接打印，不裁切 |
| 生成方式（开发参考） | 服务端 puppeteer 渲染 HTML → PDF（首选）；备选 wkhtmltopdf |

### 0.3 不是什么 / 不做什么
- ❌ 不是 ERP，不管订单、库存、客户。
- ❌ 不是财务系统，不做对账、记账、税务申报。
- ❌ 不接入 Google 表格 / 数据库（V1.0 全部手动输入金额，接口预留）。
- ❌ 不发邮件、不在线签章、不审批。
- ❌ 不出 Excel / Word / 图片，**只出 PDF**。
- ❌ 不做登录 / 多用户 / 权限（V1.0 内部单人用，可选简易密码）。
- ❌ 不做移动端（V1.0 仅桌面端 Chrome / Edge，最小宽度 1280px）。

### 0.4 主用户画像
- **运营 / 财务（朱逸骁 Eason）**：每月 1-5 号集中开票，每月一次，每次 8-20 家店铺。
- 技术水平：会用 Excel、Google 表格，不会写代码。
- 痛点：手动改 Excel 金额 + 另存 PDF，30+ 分钟，易错。
- 期望：5 分钟内批量出完所有 PDF。

## 1. 业务规则与约束

### 1.1 日本消费税计算口径（重要）
- 税率：**10%**（V1.0 写死，不做配置）
- 用户输入：**含税金额**（消費税込）
- 计算公式：
  ```
  税抜金额 = round(含税金额 / 1.1)       ← 四舍五入到整数日元
  消费税   = 含税金额 − 税抜金额          ← 保证三数之和 = 含税金额
  ```
- 例：含税 ¥10,000 → 税抜 ¥9,091，消费税 ¥909（9091+909=10000 ✓）
- 单位：日元，**整数**，不显示小数点。
- 显示：千分位逗号 + `¥` 前缀，例 `¥1,234,567`。

### 1.2 请求书编号规则
- 格式：`{YYYYMM}-{店铺缩写}-{序号:02}`
- 例：`202604-STA-01`（2026 年 4 月 + Starry + 第 1 张）
- V1.0：同一店铺同一月份只允许 1 张请求书，序号固定 `01`。
- 如果重复生成同月同店铺，系统提示「已存在编号 XXX，是否覆盖？」。

### 1.3 月份规则
- 默认值：**上个月**（如今天 2026-05-09，默认 `2026年4月`）。
- 用户可手动改其他月份。
- 选择月份 = 案件名年月 = 请求书编号年月，三者联动。

### 1.4 请求日规则
- 默认值：**今天**（如 2026-05-09 → 显示 `2026年5月9日`）。
- 用户可改任意日期。
- 与"月份"无关（月份指业务月份，请求日指开票日期）。

### 1.5 金额边界
- 最小：`¥1`
- 最大：`¥99,999,999`（约 1 亿日元）
- 必须为正整数，不允许 0 或负数。
- 输入小数自动截断为整数（带 toast 提示）。

### 1.6 数量规则
- V1.0 写死「1式」，**不允许修改、不显示输入框**。
- 后续版本如需多明细行再扩展。

## 3. 核心工作流（用户视角）

```
每月 1-5 号，运营登录 →

  [步骤1] 进入「请求书生成」页（默认页）
       ↓
  [步骤2] 选择月份（默认上月）+ 勾选要开票的店铺（默认全选）
       ↓ 点击「加载店铺」
  [步骤3] 表格列出每家店铺一行，依次填入"含税金额"
          → 输入瞬间联动显示税抜 / 消费税
          → 单行可点「预览」查看 PDF 样式
       ↓
  [步骤4] 检查无误后点「批量生成 PDF」
       ↓
  [步骤5] 系统按 Excel 模板渲染 N 份 PDF
          → 单份 → 1 个 .pdf 下载
          → 多份 → 1 个 .zip 下载
          → 同时入「历史记录」库
       ↓
  历史记录页可随时回看、重新下载 PDF
```

**性能目标**：从打开系统到下载到 ZIP，**10 家店铺 ≤ 5 分钟**（含人工输入金额时间）。

## 9. 校验规则与错误文案（中文系统 UI）

| 场景 | 触发 | 文案 |
| --- | --- | --- |
| 金额为空 | 失焦时空值 | （不报错，仅禁用生成按钮） |
| 金额非整数 | 输入 `123.45` | 自动截断为 `123` + toast「金额已自动取整」 |
| 金额超上限 | > 99,999,999 | 红边 + 提示「金额最大 ¥99,999,999」 |
| 金额 ≤ 0 | 输入 0 或负 | 红边 + 提示「金额必须大于 0」 |
| 缩写重复 | 新建店铺时 | 红字「缩写 STA 已被 starry 占用」 |
| 必填缺失 | 提交时 | 字段红边 + 「请填写{字段名}」 |
| 编号重复 | 同店同月已存在 | 弹框「2026年4月 已为 starry 生成 202604-STA-01，是否覆盖？」 |
| 导入失败 | Excel 解析错 | 弹框列出错误行 + [ 修正后重新上传 ] |
| 网络失败 | 生成 PDF 时 | toast 红「生成失败，请检查网络后重试」+ [重试] |
| 浏览器不支持 | 非 Chrome/Edge | 顶部黄条「建议使用 Chrome / Edge 获得最佳体验」 |
